# ClaimSense — Navigation & Button Hover Fixes

## Kontext
Tento prompt opravuje konkrétní hover stavy a navigační chování. Neměň nic, co není explicitně uvedeno.

---

## OPRAVA 1 — Pricing: "From €3/claim" button — outline styl

Tlačítko "From €3/claim" (Quick Start) a "From €5/claim" (Full Scale) tvoří vizuální pár — jsou navzájem swapem:

```
€5 DEFAULT  →  oranžový filled       ←→   €3 HOVER
€5 HOVER    →  bílý + oranžový okraj ←→   €3 DEFAULT
```

```
€3 DEFAULT:  bílý podklad + oranžový border (3px solid #FF5539) + oranžový text (#FF5539)
€3 HOVER:    oranžový podklad (#FF5539) + černý text (#000000)
```

```css
.btn-quickstart {
  background-color: #ffffff;
  color: #FF5539;
  border: 3px solid #FF5539;
  box-shadow: none;
  transition: background-color 0.2s ease, color 0.2s ease;
}

.btn-quickstart:hover {
  background-color: #FF5539;
  color: #000000;
  border: 3px solid #FF5539;
  box-shadow: none;
}
```

Tlačítko "From €5/claim" (Full Scale karta) zůstává beze změny — funguje správně.

## OPRAVA 2 — Pricing subpage: Optional add-ons sekce — hover nefunguje

Najdi tlačítka v sekci "Optional add-ons" na pricing stránce. Aplikuj standardní hover pravidlo:

```css
/* Pro všechna tlačítka v Optional add-ons sekci */
.optional-addons .btn,
.optional-addons button,
[class*="addon"] .btn {
  transition: border-color 0.2s ease, background-color 0.2s ease, color 0.2s ease;
}
```

Hover stav: stejný jako PRAVIDLO 1 nebo PRAVIDLO 2 z master CTA systému podle toho, jaký mají aktuální podklad (oranžový nebo bílý).

## OPRAVA 5 — Top menu: hover na navigačních linkách nefunguje

Aktuální chování: barva odkazu se změní na oranžovou pouze když je uživatel NA dané stránce (active stav). Při pouhém najetí myší se nic neděje.

Správné chování:
- **Hover**: text se změní na oranžovou (`#FF5539`)
- **Active** (aktuální stránka): text zůstane oranžový trvale

```css
/* Hover stav — při najetí myší */
nav a:hover,
.nav-link:hover,
header nav a:hover {
  color: #FF5539;
  transition: color 0.2s ease;
}

/* Active stav — aktuální stránka */
nav a.active,
nav a[aria-current="page"],
.nav-link.active {
  color: #FF5539;
}

/* Platí i pro položky v Product dropdown menu */
.nav-dropdown a:hover {
  color: #FF5539;
  transition: color 0.2s ease;
}
```

## OPRAVA 6 — Footer: obnov oranžové zvýraznění aktuální stránky

Před předchozími úpravami se ve footeru zobrazovala oranžová barva u odkazu odpovídajícího aktuální stránce (např. na Pricing stránce byl odkaz "Pricing" ve footeru oranžový). Toto chování obnov:

```css
/* Footer — aktuální stránka */
footer a.active,
footer a[aria-current="page"],
.footer-nav a.active {
  color: #FF5539;
}

/* Footer — hover stav */
footer a:hover,
.footer-nav a:hover {
  color: #FF5539;
  transition: color 0.2s ease;
}
```

Poznámka: pokud je aktivní stav řízen JavaScriptem (přidávání třídy `.active` podle aktuální URL), zkontroluj že tento JS stále funguje a třída se správně přidává i na footer odkazy.

## OPRAVA 7 — Odstranění oranžového rectangle okolo navigačních odkazů

Rectangle se neobjevoval před úpravami CTA tlačítek — není to browser default, ale CSS bug. Při přidávání border systému pro tlačítka se border nebo outline pravidlo přelilo i na navigační odkazy které border mít nemají.

Oprav pomocí přesného selektoru který cílí pouze na odkazy BEZ třídy `.btn` nebo `.pill`:

```css
/* Navigační a footer linky — border patří pouze .btn elementům, ne linkám */
nav a:not([class*="btn"]):not([class*="pill"]),
header a:not([class*="btn"]),
footer a:not([class*="btn"]) {
  border: none !important;
  outline: none !important;
  box-shadow: none !important;
}
```

Toto pravidlo se nedotkne CTA tlačítek ani pills — jejich border systém zůstane beze změny.
